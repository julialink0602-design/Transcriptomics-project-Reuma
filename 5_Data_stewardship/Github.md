## 📁 Github

In mijn project heb ik GitHub gebruikt als omgeving voor het beheren, documenteren en delen van alle projectbestanden. GitHub fungeerde als mijn **data stewardship-platform**, waarin ik structuur aanbracht, versiebeheer toepaste en reproduceerbaarheid waarborgde.

Ik heb een **duidelijke mappenstrucutuur** opgezet met aparte folder voor scripts, resultaten, data, bronnen, achtergrondinformatie en methodedocumentatie. Hierdoor is het voor mij en voor anderen direct zichtbaar wat er in elke map staat. Ook heb ik de bestandsnamen consequent en beschrijvend gemaakt, zodat je zonder openenen kunt zien wat de inhoud is.

Met behulp van **versiebeheer** op GitHub kon ik elke wijziging in mijn R-scripts volgen. Dit maakt het mogelijk om eerdere versies terug te halen en precies te zien welke stappen zijn aangepast. Dit is essentieel voor reproduceerbaarheid, omdat iemand anders mijn analyse exact kan herhalen met dezelfde code.

Mijn README-pagina beschrijft het volledige flowschema, van alignen tot DES2-analyse, GO/KEGG-analyse en IL-17-pathway visualisatie. Door deze documentatie kan een andere onderzoeker mijn project eenvoudig begrijpen en uitvoeren. Het flowschema en de figuren die ik heb toegevoegd maken de structuur nog duidelijker.

Github draagt zo bij aan **reproduceerbaarheid en toegankelijkheid** van mijn onderzoek. Alles is overzichtelijk terug te vinden, logisch georganiseerd en voorzien van uitleg, waardoor mijn project voldoet aan de principtes van goede data stewardship.
